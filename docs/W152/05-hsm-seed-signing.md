**TL;DR** — For a small studio (~10K signs/day), YubiHSM 2 (~$650) or Google Cloud HSM (~$8/month) is the economic sweet spot, while operator scale (~10M/day) needs Thales Luna A750/A790 (10K–20K ECC signs/sec) or an AWS CloudHSM cluster. No slot regulator yet mandates FIPS 140-3, but FIPS 140-2 certificates expire in September 2026, so new procurements should target 140-3.---### Key Findings- **FIPS 140-3 sunset is real.** NIST will move all FIPS 140-2 certificates to the Historical List on 21 September 2026. Thales Luna 7 (K7/G7 modules, cert #4684/#4962) and Entrust nShield 5 (cert #4745) are already FIPS 140-3 Level 3 validated; Utimaco SecurityServer Se Gen2 and YubiHSM 2 remain FIPS 140-2 Level 3. AWS CloudHSM and Google Cloud HSM are FIPS 140-2 Level 3 as of current vendor disclosures. (NIST CMVP IUT List, 2026; Thales Docs, 2025; Entrust nShield 5 Security Policy, 2024)- **Algorithm coverage varies.** All six support ECDSA P-256 and RSA-4096. Ed25519 is supported by Luna 7, nShield 5, YubiHSM 2, and AWS CloudHSM. Google Cloud HSM does *not* list Ed25519—only EC P-256/P-384 and RSA-4096. (Thales Luna Specs; Yubico Product Page; Google Cloud KMS Docs; AWS CloudHSM Algorithm Guide)- **Throughput spans four orders of magnitude.** Luna A790 peaks at 20,000 ECC P-256 signs/sec; Utimaco Se Gen2 at ~6,000 ECDSA signs/sec; AWS CloudHSM hsm2m.medium at ~2,500 RSA-2048 signs/sec. YubiHSM 2 manages roughly 13 ECDSA-P256 signs/sec (~73 ms/op). Google Cloud HSM reports 20–50 ms latency per EC sign, implying ~20–50 ops/sec single-threaded. (Thales Product Brief; Utimaco Se Gen2 Datasheet; AWS Certification Handbook; Yubico Performance Table; OneUptime Blog)- **Rust integration is straightforward for PKCS#11 devices.** The `rust-cryptoki` crate (Parallaxsecond) wraps PKCS#11 and is confirmed compatible with Luna, AWS CloudHSM, YubiHSM, Utimaco, and nShield. Google Cloud HSM has no PKCS#11 provider; it requires gRPC/REST via the `google-cloud-kms` crate. (GitHub rust-cryptoki changelog; NLnet Labs Cascade docs)- **Cost at small-studio volume:** YubiHSM 2 is a one-time ~$650 capex. Google Cloud HSM is ~$6/key/month plus ~$0.03/10,000 ops, yielding roughly $8–10/month at 10K signs/day. AWS CloudHSM costs ~$1.45/hour per instance (~$2,088/month minimum for HA). (Crypto-Store; QCEcuring Blog; AWS Certification Handbook)- **Cost at operator scale:** 10M signs/day on Google Cloud HSM would run ~$900/month in operations alone, but latency would accumulate. A Luna A750 (10K ECC signs/sec = 864M/day) or AWS CloudHSM multi-instance cluster is the operator-grade path. (Thales Specs; AWS Certification Handbook)- **Regulators focus on RNG randomness and auditability, not HSM model.** UKGC RTS 7, MGA technical compliance, and GLI-19 Chapter 4 require independent lab certification, source-code review, statistical testing, and seed unpredictability. None explicitly mandates FIPS 140-3 or a specific HSM brand, but AGCO and UKGC guidance strongly imply hardware-backed key protection for seed integrity. (UKGC RTS PDF; GLI-19 v3.0; AGCO Standards PDF)---### Contradictions / Open Questions- **Cloud HSM acceptability for gaming seeds is unresolved.** Vendor docs and cloud architects treat AWS CloudHSM/Google Cloud HSM as equivalent to on-prem FIPS 140-2/140-3 Level 3, yet gaming test labs and some regulators historically prefer customer-controlled physical appliances for RNG root keys. No explicit prohibition exists, but procurement risk remains until a major operator secures a UKGC/MGA sign-off with a cloud-native HSM.- **FIPS 140-3 deadline vs. gaming industry inertia.** FIPS 140-2 expires in September 2026, but gaming labs and regulators are not yet demanding 140-3 for slot RNGs. The delta creates a procurement trap: buying 140-2-only hardware today may force a replacement cycle within 12–24 months.---### Practical Implications for Boki / Cortex- **Small studio / dev:** Start with YubiHSM 2 for dev signing and Google Cloud HSM for production if you’re already on GCP and can tolerate REST-based signing. Avoid AWS CloudHSM at this volume—it is economically irrational.- **Operator scale / multi-jurisdiction:** Target Thales Luna A750 or A790 (FIPS 140-3, PKCS#11, 10K–20K signs/sec) for on-prem or Luna Cloud HSM/DPoD if you need cloud elasticity with a familiar PKCS#11 surface. If AWS-native, run a CloudHSM cluster with `rust-cryptoki`.- **Rust slot engine:** Use `rust-cryptoki` for Luna/CloudHSM/YubiHSM/Utimaco/nShield. For Google Cloud HSM, wrap the gRPC client in a small async signer service to isolate the non-PKCS#11 path.---### Sources1.
