# TLS Configuration — SOC 2 Evidence

CORTI W206-SECURITY — TLS termination evidence collection for the
SOC 2 Type 1 audit. Closes OWASP A02 high-severity finding
`a02-nginx-no-tls`.

Source of truth: `docker/nginx-spa.conf`.

## Architecture

```
Internet ──HTTPS:443──> nginx (SPA + reverse proxy) ──HTTP:4000──> server
                            │
                            └─ TLS 1.3 termination
                            └─ HSTS / X-Frame-Options / CSP fan-out
```

The Fastify backend itself listens on plain HTTP inside the bridge
network only — nginx is the single TLS entry point. This pattern
matches the AWS ALB / GCP HTTPS LB / Azure AppGw blueprint and isolates
cert rotation from the server image.

## Configured directives

| Directive | Value | Rationale |
|-----------|-------|-----------|
| `listen 443 ssl` + `http2 on` | Enabled | HTTP/2 over TLS only |
| `ssl_protocols` | `TLSv1.3` | PCI-DSS 4.0 § 4.2.1, NIST SP 800-52r2 § 3.1 |
| `ssl_prefer_server_ciphers` | `off` | TLS 1.3 negotiates AEAD by spec |
| `ssl_session_cache` | `shared:SSL:10m` | 10 MB cache → ~40 k sessions |
| `ssl_session_timeout` | `1d` | Balance perf vs forward secrecy |
| `ssl_session_tickets` | `off` | Avoid stateful ticket leakage |
| `ssl_stapling` | `on` | OCSP must-staple |
| `ssl_stapling_verify` | `on` | Detect tampered OCSP responses |
| `resolver` | `1.1.1.1 8.8.8.8` | OCSP responder DNS |

## HTTP → HTTPS redirect

Plaintext :80 returns `301` to `https://$host$request_uri`. Exception:
the `/.well-known/acme-challenge/` path is served directly so Let's
Encrypt HTTP-01 renewals don't break.

## Security headers attached at edge

| Header | Value |
|--------|-------|
| Strict-Transport-Security | `max-age=31536000; includeSubDomains; preload` |
| X-Frame-Options | `SAMEORIGIN` |
| X-Content-Type-Options | `nosniff` |
| Referrer-Policy | `strict-origin-when-cross-origin` |
| Permissions-Policy | `geolocation=(), microphone=(), camera=()` |

These mirror the Helmet headers attached at the Fastify layer — see
`docs/SOC2_EVIDENCE/SECURITY_HEADERS.md`.

## Certificate management

- **Production:** Let's Encrypt fullchain + privkey mounted into
  `/etc/nginx/certs/` via the `docker-compose.yml` `volumes:` block on
  each SPA service (`studio`, `operator`, `regulator`).
- **Development:** A self-signed pair is generated by the SPA image
  entrypoint before nginx starts (see `Dockerfile.*` `generate-selfsigned`
  step planned in a follow-on wave).
- **Rotation:** `certbot renew` runs nightly via a sidecar; on success
  it triggers `nginx -s reload` (no downtime). Targeted RPO/RTO < 30 d
  (Let's Encrypt cert lifetime is 90 d; alert fires at 30 d remaining).

## Verification commands

```bash
# Verify the certificate chain
openssl s_client -connect operator.example.com:443 -showcerts < /dev/null

# Confirm only TLS 1.3 is negotiated
openssl s_client -connect operator.example.com:443 -tls1_3 < /dev/null
openssl s_client -connect operator.example.com:443 -tls1_2 < /dev/null
# ↑ should fail with "no protocols available"

# Inspect security headers
curl -sI https://operator.example.com/ | grep -i strict-transport
```
