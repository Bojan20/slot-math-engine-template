# Internationalization (i18n)

> **Status:** W208 Faza 400.1 — Bootstrap (en / es / pt / de).
> **Module:** `web/studio/src/i18n/`.
> **Owner:** Studio frontend team.

The Studio ships with a vanilla-TypeScript i18n core (zero external
dependencies). We own 100% of the surface so that locale metadata can
flow directly into the cert paper trail without fighting upstream
library opinions.

## Supported locales

| Code | Flag | Native name | Status |
|---|---|---|---|
| `en` | 🇬🇧 | English | Source of truth |
| `es` | 🇪🇸 | Español | W208 seed (full parity) |
| `pt` | 🇧🇷 | Português | W208 seed (full parity, BR) |
| `de` | 🇩🇪 | Deutsch | W208 seed (full parity) |

RTL languages (Arabic, Hebrew, Persian, Urdu) are **infrastructure-ready**
via `web/studio/src/i18n/rtl.ts` — `isRtlLocale()` + the
`--text-align-start` CSS variable flip on `<html dir>`. Seed
translations land in a later W2xx wave.

## Public API

```ts
import { t, setLocale, getCurrentLocale } from '../i18n/index.js';

t('nav.build');                                 // "BUILD"
t('errors.validation', { reason: 'bad' });      // "Validation failed: bad"
t.plural('items', 5);                           // "5 items" (Intl.PluralRules)
t.num(1234.5);                                  // "1,234.5" (en) / "1.234,5" (de)
t.currency(99.5, 'USD');                        // locale-aware $ format
t.date(new Date(), { dateStyle: 'long' });      // locale-aware date

setLocale('es');                                // persists to localStorage
getCurrentLocale();                             // 'es'
```

## Adding a new key

1. Open `web/studio/src/i18n/locales/en.ts`.
2. Add the new entry in the matching namespace (`nav` / `actions` /
   `symbols` / `metrics` / `modals` / `errors` / `tooltips` /
   `producer`).
3. Mirror it in `es.ts`, `pt.ts`, `de.ts`. TypeScript will refuse to
   compile the locale file if any key is missing — that is the parity
   contract.
4. Run `npx vitest run tests/i18n-w208.test.ts` — the parity matrix
   assertion will also flag missing keys at test time.

## Adding a new locale

1. Create `web/studio/src/i18n/locales/<code>.ts` exporting a
   `TranslationDict` (full key set, no omissions).
2. Add `<code>` to the `Locale` union in `types.ts` and to
   `SUPPORTED_LOCALES` + `LOCALE_DISPLAY` (flag + native name).
3. Import + register in `index.ts` (`DICTS` map).
4. If RTL, add the base tag to `RTL_BASE_TAGS` in `rtl.ts`.
5. Extend the test parity matrix at
   `web/studio/tests/i18n-w208.test.ts`.
6. Run the studio test suite — every assertion must stay green.

## Naming conventions

- **Namespace prefix:** `nav.` / `actions.` / `symbols.` / `metrics.` /
  `modals.` / `errors.` / `tooltips.` / `producer.`.
- **dot.snake_case** within namespaces (`modals.new_game.title`).
- **Placeholders:** `{name}` style (ICU-flat). Keep param names
  identical across locales (the same `{name}` should be replaceable
  one-to-one).
- **Plurals:** create sub-keys `<base>.one` / `<base>.other` (plus
  language-specific `few` / `many` / `zero` when needed). Call via
  `t.plural(base, count)`.

## Translator workflow

1. Translator works in the language-specific `.ts` file. The keys are
   `dot.snake_case` and stable across releases.
2. Reviewer runs `npx tsc --noEmit -p tsconfig.json` to catch missing
   keys (TypeScript narrows `TranslationDict` against the English
   source).
3. Acceptance: `npx vitest run tests/i18n-w208.test.ts` must pass
   (parity matrix + per-locale empty-string assertion).
4. Visual QA: launch the Studio (`npm run dev`), use the
   LocaleSwitcher dropdown in the header to flip languages and review
   the BUILD / COMPOSE / CATALOG tabs.

## RTL plan (future W2xx)

Arabic / Hebrew / Persian / Urdu translations land alongside their
respective LATAM/EMEA market launches. The infrastructure is already
in place:

- `isRtlLocale()` recognises the base tags.
- `applyDirectionToDocument()` flips `<html dir>` + the
  `--text-align-start` / `--text-align-end` CSS variables.
- Studio CSS should consume `text-align: var(--text-align-start, left)`
  rather than hard-coded `text-align: left`.

When the first RTL locale ships, also audit:
- Icon/arrow flip semantics (chevrons in nav).
- Mirrored padding/margin (use logical `padding-inline-start` where
  practical).
- Number formatting (Arabic numerals are LTR even inside RTL text).

## Persistence

The selected locale is persisted to `localStorage` under the key
`slot-math-studio.locale`. On boot the i18n module reads the stored
value (if present and supported) and updates the `<html lang>`
attribute. Falls back to `en` silently when storage is unavailable
(private-mode browsers, file:// contexts).
